# スキル: 社内通知・お知らせ作成

## このスキルのゴール

読まれて、理解されて、行動につながる社内通知を作る。
「一応送りました」ではなく「全員が読んで動いた」状態を目指す。

## 社内通知の種類と特性

| 種類 | 目的 | 長さの目安 |
|------|------|-----------|
| 緊急通知 | 即時対応を求める | 3行以内 |
| 制度変更・ルール変更 | 変更内容と適用日を伝える | 200〜400字 |
| 行事・イベント案内 | 日程・場所・持ち物を伝える | 200〜300字 |
| 依頼・お願い | 具体的な行動を促す | 150〜300字 |
| 報告・共有 | 情報を周知する | 200〜500字 |

## 必ず含める要素

1. **件名**: 一言で内容がわかる（「〇〇のお知らせ」「【重要】〇〇について」）
2. **要点3行**: 件名の次に「何を・いつまでに・どうするか」を先に書く
3. **詳細**: 必要な情報の詳細（背景・理由・手順）
4. **アクション**: 読んだ後に何をすれば良いかを明示
5. **期限**: いつまでに対応が必要か
6. **問い合わせ先**: 担当者・連絡先

## 件名の書き方

| シーン | 件名の例 |
|--------|---------|
| 緊急 | 【緊急】〇〇について今すぐご確認ください |
| 重要 | 【重要】〇〇の変更について（〇月〇日適用） |
| 通常 | 〇〇についてのお知らせ |
| 依頼 | 【ご協力お願い】〇〇の対応について |
| 案内 | 【開催案内】〇〇（〇月〇日） |

## 出力フォーマット

```
件名: 〇〇についてのお知らせ

〇〇各位

（用件を3行で要約）

---

【概要】
（変更内容・事項の詳細）

【適用日・期限】
〇〇年〇〇月〇〇日（〇）

【対応いただきたいこと】
1. 〇〇してください
2. 〇〇の場合は〇〇してください

【問い合わせ先】
担当: 〇〇（〇〇部）
連絡先: 〇〇@〇〇.co.jp / 内線 〇〇〇

以上、よろしくお願いいたします。
```

## 品質チェックリスト

- [ ] 件名だけで「自分には関係あるか」判断できるか
- [ ] 最初の3行で要点が把握できるか
- [ ] 「何をすべきか」が明確か（対応不要の場合もその旨を記載）
- [ ] 期限が具体的な日時で書かれているか

## NG例

❌ 「平素より大変お世話になっております」から始まる社外向けの文体
❌ 期限が「速やかに」「できるだけ早く」等の曖昧表現
❌ 全員向けなのに特定の人しか関係ない情報を長々と書く
