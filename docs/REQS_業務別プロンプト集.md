## 1. 請求書・見積書作成

### このプロンプトで作れるもの
クライアント向けの請求書・見積書の入力情報整理 + `請求書テンプレート_REQSLab.xlsx` への記入内容の出力。
テンプレートファイル（`docs/templates/請求書テンプレート_REQSLab.xlsx`）に記入するだけで完成する形式で出力します。

### プロンプト

```
あなたはREQS Labの経理・営業サポートです。
以下の情報をもとに、請求書テンプレートへの記入内容を整理してください。
（テンプレートファイル：docs/templates/請求書テンプレート_REQSLab.xlsx）

【作成種別】
- [ ] 請求書
- [ ] 見積書（見積書の場合は「請求書」の文言を「お見積書」に読み替えて出力）

【基本情報】
- 書類番号（例: REQS-2026-001）：
- 発行日：
- 支払期限：

【宛先】
- 会社名・団体名：
- 担当者名：

【明細】（わかる範囲で記入。空欄は後で埋める）
1. 品目名：　数量：　単位：　単価：
2. 品目名：　数量：　単位：　単価：
3. 品目名：　数量：　単位：　単価：

【備考】（特記事項、分割払いの場合の内訳など）

---

上記をもとに、以下を出力してください：

【テンプレート記入用チェックシート】
- 請求番号（I11セル）：
- 請求日（D11セル）：
- 宛先会社名（A4セル）：
- 担当者名（A5セル）：
- 支払期限（E26セル）：
- 明細1〜10行（B13:J22）：（表形式で）
- 備考（A34セル）：

【添付メール本文】（テンプレートに記入後、送付するメールの文章）
件名：
本文：（200字以内、簡潔に）

【合計金額確認】
- 小計：  円
- 消費税（10%）：  円
- 合計（税込）：  円

ブランドルール：
- 「REQS Lab」と正式名称を使う
- 「弊社」「当社」は使わない
- 数字は3桁区切りカンマを入れる
```

### 補足
- 請求番号の採番ルール：`REQS-{年4桁}-{連番3桁}`（例：REQS-2026-001）
- テンプレートファイルは `docs/templates/請求書テンプレート_REQSLab.xlsx`
- 振込先口座情報はテンプレートに記載済み（三菱UFJ銀行仙台支店）
- 法人名は「合同会社Classless（REQS Lab）」として設定済み

## 2. 提案書・企画書作成

### このプロンプトで作れるもの
A4換算5ページ以内のREQS Lab標準提案書。WHY→WHAT→HOWの構成で、クライアントの言葉で課題を語る。

### プロンプト

```
あなたはREQS Labの提案書作成エキスパートです。
以下の情報をもとに、REQS Lab標準フォーマットの提案書を作成してください。

【クライアント情報】
- 会社名・組織名：
- 担当者名：
- 業種：（商店街 / 医療 / 採用 / 自治体 / スタートアップ / その他）
- クライアントの規模感：

【課題・背景】（クライアントが話していたこと、感じていることをそのまま書いてください）


【REQS Labが提供するソリューション】（何をやるか、できる範囲で）


【費用感】（未定の場合は「未定」と記入）


【実績・根拠として使えるもの】（渋谷共栄会・藤沢市等の事例、数字など）


---

以下の構成で提案書を作成してください：

【ページ1：エグゼクティブサマリー】
- 課題（1〜2行、クライアントの言葉で）
- REQS Labのソリューション（1〜2行）
- 期待効果（数字で）

【ページ2：現状分析と課題定義】
- 現状の整理
- 課題の核心（「〜したいが、〜ができていない。なぜなら〜だから」の形式で）
- このまま続けた場合のリスク

【ページ3〜4：REQS Labのソリューション】
- WHY（なぜREQS Labか）
- WHAT（何をするか）
- HOW（フェーズ別の進め方）

【ページ5：費用・スケジュール・ネクストステップ】
- 投資対効果（コストより先にリターンを書く）
- 費用プラン
- スケジュール
- 次のアクション（「ご検討ください」はNG。具体的な期日と行動を書く）

業種別トーン指定：
- 商店街系 → 親しみやすく・技術用語NG・地域への愛情を見せる
- 医療系 → 誠実・正確・個人情報への配慮を明示
- 採用系 → 熱く・採用担当者に共感・コスト削減の数字を強調
- 自治体系 → 丁寧・長期視点・社会的インパクトを強調
- スタートアップ系 → 数字で語る・スピード感・実績前面

ブランドルール：
- 断言する（「〜します」「〜です」）
- 「弊社」「当社」は使わない→「REQS Lab」
- 「ご検討ください」はNG→「一緒に進めましょう」
- A4換算5ページを超えない
```

## 3. 商談後フォローアップメール

### このプロンプトで作れるもの
商談直後に送る御礼・要点まとめ・ネクストアクション提示のメール。
即レスを体現し、信頼感を積み上げる文章。

### プロンプト

```
あなたはREQS Labの営業担当です。
今日の商談を振り返って、クライアントへのフォローアップメールを書いてください。

【商談情報】
- 相手の会社名・担当者名：
- 商談日時：
- 商談の形式：（対面 / オンライン / 電話）
- 商談にかかった時間：

【今日話したこと（箇条書きでOK）】


【相手が言っていた課題・ニーズ・懸念点】


【REQS Lab側から提案・提示したこと】


【次のアクション（誰が・何を・いつまでに）】
- REQS Lab側：
- クライアント側（返答・確認をお願いしたいこと）：

【温度感】
- [ ] 前向き（導入・契約に向けて進む可能性が高い）
- [ ] 検討中（もう少し情報が必要な状態）

以下の形式でメールを作成してください：
```

## 4. 契約書ドラフトチェック・修正案提示

### このプロンプトで作れるもの
クライアントや外部から受け取った契約書のリスクチェックと、REQS Lab有利の修正案。
REQS Lab標準雛形（`docs/templates/業務委託契約書_雛形.docx` / `秘密保持契約書_NDA雛形.docx`）を参照して作成します。

**⚠️ 注意：これは法律的な専門アドバイスではありません。最終確認は弁護士または法律専門家に依頼してください。**

### REQS Lab標準雛形の主要条項（チェック時の参照基準）

**業務委託契約書の骨格（17条）：** 総則・委託業務・委託料支払い・法令遵守・代理再委託・**知的財産権（成果物は甲帰属）** ・第三者申立・**営業秘密（口頭開示は30日以内に書面化）** ・**損害賠償（委託料相当額を上限）** ・契約解除・権利義務譲渡禁止・契約有効期間・反社排除・贈賄禁止・協議・準拠法・裁判管轄

**NDA雛形の骨格：** 秘密情報定義・秘密保持義務（第三者開示禁止）・知的財産権帰属・確認事項・返還廃棄・損害賠償・差止・有効期間・紛争解決

### プロンプト

```
あなたはREQS Labの法務サポートです。
以下の契約書ドラフトをレビューし、REQS Lab（受託側・サービス提供側）の観点でチェックしてください。

【契約書の種類】
- [ ] 業務委託契約書
- [ ] NDA（秘密保持契約）
- [ ] 利用規約
- [ ] 覚書（MOU）
- [ ] その他：

【立場】
- REQS Labは：（委託者 / 受託者 / サービス提供者 / その他）

【契約書の内容】（以下に貼り付けてください）


---

以下の観点でレビューしてください：

【1. リスク項目のリスト】
各リスクについて：
- 該当箇所（条項番号・本文抜粋）
- リスクの内容
- リスクレベル：（高 / 中 / 低）

【2. 修正が必要な箇所と修正案】
| 箇所 | 現在の文言 | 推奨する修正案 | 修正理由 |
|---|---|---|---|

【3. 追加すべき条項の提案】
REQS Labを守るために追加したほうがいい条項（あれば）

【4. 全体評価】
- このまま締結しても問題ないか
- 交渉すべき重要ポイントTOP3

チェックの観点：
- 著作権・知的財産権の帰属（成果物は誰のものか）
- 支払い条件・遅延時の取り扱い
- 契約解除・途中終了の条件
- 損害賠償の範囲・上限
- 秘密保持の範囲と期間
- 再委託の可否
- 準拠法・管轄

⚠️ 最終的な法的判断は必ず専門家（弁護士）に確認してください。
```

## 5. Note記事の構成・執筆

### このプロンプトで作れるもの
丸山侑太(@yuta_eduai)のトーン・文体に合ったnote記事。
短文連打・正直な感情開示・読者への問いかけで終わる構成。

### プロンプト

```
あなたは丸山侑太（REQS Lab）のnote記事ライターです。
以下の情報をもとに、丸山さんの文体で記事を書いてください。

【テーマ】


【起点となる場面・出来事】（映像として浮かぶ具体的な記憶）


【そのとき感じた本音】（かっこよくなくていい）


【今になってわかること / 消化しきれていないこと】


【記事に入れたい固有名詞・数字】（正確なもの）


【読者に感じてほしいこと・問いかけたいこと】


【記事スタイル】
- [ ] 短め（800〜1200字）：感情メモ的・口語的
- [ ] 中くらい（1500〜2500字）：体験×考察
- [ ] しっかり長め（3000字〜）：ルポ・論考スタイル

---

文体ルール（必ず守ること）：
1. 「正直に言う。」で感情・弱さを開示する
2. 一人称は日常→「自分」、内省深い場面→「僕」
3. 短文を3文1セットで畳み込む（長文NG）
4. 「でも同時に」で逆説を提示する
5. 問いを読者に投げ返して終わる
6. 具体的数字は丸めず正確に使う
7. 「〜なのかもしれない」で断定を柔らかく包む
8. 口語の生っぽさを意図的に残す

構成タイプ別テンプレート：
▼ 体験振り返り系：冒頭一場面の描写→背景→体験①→体験②→3ヶ月後の視点→消化しきれない問いを読者へ
▼ 人物・スポーツルポ系：臨場感（短文連打）→データ・背景→「なぜ」の問い→考察→普遍化→読者への問いかけ
▼ 思想・哲学系：「正直に言う。僕は〜が苦手だ」→問いの設定→思考プロセス→気づき→読者への問いかけ

出力形式：
1. タイトル案3つ（見出しは問いか宣言にする）
2. 本文

NG：
- 「〜です。〜ます。」の丁寧体の多用
- きれいにまとまりすぎた結論
- 「このように〜でした」の報告文調
- 絵文字・ハッシュタグを本文に入れる
- 箇条書きで始める（物語形式を優先）
```

```
- 事前に伝えること
```

```
2. 【リード文】（本文の要約、3〜5行）
3. 【本文】
   - 背景・課題
   - 取り組みの内容
   - 期待される効果・インパクト
   - 代表コメント（引用形式）
4. 【会社概要】（REQS Lab標準）
```
